# Retail Sales & Profitability Analysis

An end-to-end data analyst portfolio project: raw data → cleaning → SQL analysis → Python EDA → interactive dashboard → business insight report.

**Dataset:** ~5,500 retail order transactions (2022–2025), schema modeled on the well-known "Sample Superstore" dataset (Order ID, Order/Ship Date, Region, Category, Sub-Category, Sales, Profit, Discount, Quantity, etc.), including realistic data-quality issues: duplicate rows, missing values, mixed date formats, and sign-entry errors — so the cleaning step is real, not cosmetic.

> Note: the raw CSV here is synthetically generated (`notebooks/01_generate_data.py`) because this build environment can't reach kaggle.com. The schema and business logic exactly match the real Kaggle "Sample Superstore" dataset, so you can drop in the real file from [kaggle.com/datasets → search "Sample Superstore"] and every downstream script runs unchanged — just replace `data/superstore_raw.csv` and re-run steps 2–4.

## What's in this repo

```
data/
  superstore_raw.csv        # raw, messy export
  superstore_cleaned.csv    # cleaned dataset (output of step 2)
  superstore.db             # SQLite DB loaded with the cleaned data
notebooks/
  01_generate_data.py       # generates the synthetic raw dataset
  02_clean_data.py          # cleaning: dedup, dates, nulls, data-entry errors
  03_eda_analysis.py        # exploratory analysis + saves charts
  04_load_sqlite.py         # loads cleaned data into SQLite
sql/
  queries.sql                # 10 business-question queries: joins, CTEs, window functions
dashboard/
  index.html                  # interactive dashboard (open directly in any browser)
charts/
  01-05_*.png                 # charts generated by the EDA script
  key_findings.txt            # plain-text summary of findings
report/
  insight_summary.docx        # formatted business insight report
```

## How to run it yourself

```bash
cd notebooks
python3 01_generate_data.py     # or skip this and drop in the real Kaggle CSV as superstore_raw.csv
python3 02_clean_data.py
python3 03_eda_analysis.py
python3 04_load_sqlite.py
```

Then open `dashboard/index.html` directly in a browser — no server needed, the data is embedded in the page. Use the Region / Category / Segment / Year filters to explore.

To run the SQL layer:
```bash
sqlite3 data/superstore.db < sql/queries.sql
```

## Key findings

1. **Technology** is the strongest category by profit (~$414K); **Furniture** is the only category that's net loss-making, driven almost entirely by **Tables**.
2. **Discounting above ~30% turns sales into losses** — average margin falls from +22.4% (≤10% discount) to -3.6% (≥30% discount).
3. **South region** leads in profit despite similar sales volume to other regions; **East** lags, suggesting a cost or discount-approval issue rather than a demand problem.
4. **Tables** is the single biggest drag on profitability at the sub-category level (-$33.5K).
5. Full detail and recommendations are in `report/insight_summary.docx`.

## Suggested talking points for interviews

- **Why this dataset/approach?** Mirrors a real business scenario (retail P&L by category/region) rather than a toy ML dataset — shows business-analysis thinking, not just modeling.
- **What was messy about the data, and how did you handle it?** Walk through `02_clean_data.py` — duplicates, mixed date formats, nulls, sign errors, and *why* each fix was the right call (e.g., imputing vs. dropping).
- **What's the single most actionable insight?** The discount-margin cliff — it's concrete, quantified, and immediately actionable by a pricing/sales team.
- **How would you extend this?** Add a real ML layer (e.g., predicting which orders are likely to be unprofitable at order-entry time), or connect the dashboard to a live database instead of a static CSV.
